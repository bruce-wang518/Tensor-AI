文档概述

除登录接口外，所有接口都需要在Header中提供token和appid，无效token会返回401未授权错误。

示例域名：文档中的 localhost:8080 需替换为实际部署域名

统一响应格式

所有接口都遵循以下JSON格式返回数据：

{
    "code": 0,       // 0=成功，非0=错误
    "message": null, // 成功时为null，错误时为错误信息
    "data": {}      // 实际返回的数据内容
}

通用错误代码：

  • 401：未授权（无效token）
  • 400：请求参数错误
  • 500：服务器内部错误

1. 登录接口

用户登录获取访问令牌

POST /api/login

请求头

Content-Type: application/json

请求体

{
    "mobile": "12345678901",
    "code": "12345"
}

cURL示例

curl -X POST -H "Content-Type: application/json" -d "{\"mobile\":\"12345678901\",\"code\":\"12345\"}" http://localhost:8080/api/login

成功响应

{
    "code": 0,
    "message": null,
    "data": "token_123456"
}

2. 获取会话列表

获取当前应用的会话列表

GET /api/sessions

请求头

token: 登录返回的token

appid: 应用ID（如process, product, data等）

cURL示例

curl -H "token: token_123456" -H "appid: process" http://localhost:8080/api/sessions
